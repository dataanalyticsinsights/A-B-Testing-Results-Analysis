# A-B-Testing-Results-Analysis

# Project Overview

This project focused on improving the revenue of an online store by prioritizing business hypotheses and evaluating the results of an A/B test.

The complete workflow, including data cleaning, preprocessing, visualizations, and statistical analysis, was developed in the Project_Sprint_09_en.ipynb notebook.

# Objective

The objective of the project was to identify the most promising business hypotheses and measure the impact of implemented changes on user behavior and revenue performance.

# Methodology

The analysis included:

Data cleaning and preprocessing
Hypothesis prioritization using ICE and RICE frameworks
A/B test analysis
Statistical testing and data visualization

# Performance was evaluated using metrics such as:

Cumulative revenue

Average order value

Conversion rate

Both raw and filtered datasets were analyzed to reduce the impact of outliers.

# Key Findings

The analysis compared the performance of groups A and B to determine which version produced better business results.

# Key insights included:

Differences in revenue and conversion rates between test groups

The significant impact of outliers on overall results

More reliable conclusions after filtering abnormal transactions

# The findings helped validate business hypotheses and support data-driven decision-making.
